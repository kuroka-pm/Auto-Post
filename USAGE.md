## 画面構成

アプリは 5 つのタブで構成されています。

| タブ | 内容 |
|------|------|
| 🔑 API設定 | Gemini / X / Threads の API キーを管理 |
| 🎭 ペルソナ | 投稿の人物像を設定・AI 自動生成 |
| 📰 情報源 | トレンド取得元の RSS URL を設定 |
| ✍️ 文体スタイル | 5 種類の文体の重みと内容をカスタマイズ |
| ▶️ 実行 | 投稿の生成・プレビュー・投稿・自動運用 |

---

## 初期設定

### 1. API キーを入力

**API設定タブ** を開き、以下のキーを入力して「💾 保存」します。

#### Gemini API（必須）
- [Google AI Studio](https://aistudio.google.com/apikey) で API キーを作成
- モデルは `gemini-2.5-flash`（無料枠）が推奨
- 利用頻度が高い場合はクレジットカードの登録が必要になる場合があります

#### X (Twitter) API（X に投稿する場合）
- [Developer Portal](https://developer.x.com/en/portal/dashboard) で取得
- **Read and Write** 権限が必要
- API Key / API Secret / Access Token / Access Token Secret の 4 つ
- 無料枠（Free）では投稿が制限されるため、Basic プラン（$5/月〜）への加入が必要になる場合があります

#### Threads API（Threads に投稿する場合）
- [Meta for Developers](https://developers.facebook.com/) で取得
- User Access Token を入力

### 2. 接続テスト

「🔌 接続テスト」ボタンで各 API の接続状況を確認できます。

- ✅ 接続成功: キーが有効です
- ❌ 接続失敗: キーの値や権限を確認してください

## ペルソナの設定

**ペルソナタブ** で投稿者の人物像を設定します。

### 入力項目

| 項目 | 例 |
|------|------|
| 性別 | 女性 |
| 年齢 | 28歳 |
| 職業 | IT企業のプロジェクトマネージャー |
| 経歴 | 新卒でSIerに入社、3年目に転職... |
| 趣味 | カフェ巡り、読書、猫 |
| 性格 | 几帳面だけどたまに抜けてる |
| 一人称 | わたし |
| 口調・語尾 | 〜だよね、〜かも |

### AI でペルソナ自動生成

1. 基本情報を入力
2. 「🤖 AI でペルソナ生成」ボタンをクリック
3. AI が詳細なペルソナ設定文を自動生成
4. 必要に応じて手動で編集
5. 「💾 保存」

## 情報源（トレンド）の設定

**情報源タブ** でトレンド取得元の RSS URL を設定します。

### デフォルトで設定済みのソース

- Google トレンド（日本）
- Yahoo! ニュース
- はてなブックマーク

### カスタマイズ

- RSS URL を追加・削除可能
- カテゴリフィルタで特定ジャンルのみ取得
- ブラックリストで除外ワードを設定

## 投稿スタイルの設定

**文体スタイルタブ** で 5 種類の文体を管理します。

| スタイル | 文字数目安 | 説明 |
|----------|-----------|------|
| 本音つぶやき | 60〜100 | 短くてリアルなつぶやき |
| 共感ポエム | 100〜140 | 感情に訴える投稿 |
| ストーリー | 120〜140 | エピソードベースの投稿 |
| ニュース一言 | 60〜100 | トレンドへの短いコメント |
| まとめ・教訓 | 100〜140 | 経験からの学び |

各スタイルの **重み**（選ばれやすさ）を数値で設定できます。

## 手動で投稿する

### 基本の流れ

1. **実行タブ** を開く
2. スタイルをドロップダウンで選択（「ランダム」も可）
3. 「📝 サンプル文章を生成」をクリック
4. プレビューエリアに生成文が表示される
5. 文字数カウンターで長さを確認（140 文字超: 赤表示）
6. 必要に応じてプレビューを直接編集
7. 気に入らなければ「🔄」ボタンで再生成
8. 「🐦 この文章を投稿する」をクリック

### 投稿先の選択

実行タブの「投稿先」で X / Threads のチェックを切り替えます。

## 自動運用

### 設定

1. **実行時刻**: カンマ区切りで指定（例: `09:00, 12:00, 18:00, 21:00`）
2. **時間ゆらぎ**: 0〜n 分のランダム遅延でボット検知を回避
3. **投稿先**: X / Threads を選択

### 開始・停止

- 「▶️ 自動運用スタート」で開始
- 「⏹️ 停止」で停止
- ログエリアに実行状況がリアルタイム表示されます

---

## 投稿タイプについて

自動運用時、3 種類の投稿タイプからランダムに選択されます。

| タイプ | 内容 | デフォルト比率 |
|--------|------|---------------|
| **A: トレンド連動** | トレンドワードに反応する投稿 | 3 |
| **B: 独立意見** | トレンドに関係なくペルソナの日常を投稿 | 1 |
| **C: note 告知** | note 記事の告知投稿 | 1 |

比率は実行タブで変更可能です。

## note 告知投稿

### 設定方法

1. 実行タブの「note 記事リスト」に記事情報を追加
   - URL
   - タイトル
   - 概要（任意）
2. 自動運用でタイプ C が選ばれた時、自然な告知文を AI が生成

### 手動で note 告知

- 「📝 note 告知として生成」にチェックを入れて生成

## 画像付き投稿

1. 「🖼️ 画像を添付」で画像ファイルを選択
2. ALT テキスト（画像の説明文）を入力（任意）
3. 通常通り「📝 生成」→「🐦 投稿」

> 画像は X への投稿時のみ反映されます。

## トラブルシューティング

### 「API Key が無効です」

- API キーが正しくコピーされているか確認
- 「🔌 接続テスト」で各 API の状態を確認
- X API: 権限が **Read and Write** になっているか確認

### 投稿が失敗する

- ログエリアおよび `logs/` フォルダのログファイルを確認
- X API: Access Token を権限変更後に再生成しているか確認
- Threads API: User Access Token の有効期限を確認

### 生成されるテキストが不自然

- ペルソナ設定を詳しく書く（具体的なエピソードや数字を含める）
- 「一人称」「口調・語尾」を明示的に設定
- 文体スタイルの手本文を見直す

### exe が起動しない（セキュリティ警告）

- **SmartScreen**: 「詳細情報」→「実行」をクリック
- **Smart App Control**: 設定 → Windows セキュリティ → アプリとブラウザ制御 → Smart App Control → オフ

## ログファイル

ログは自動的に `logs/YYYY-MM-DD.log` に保存されます。
アプリを閉じた後でも、過去の実行履歴やエラーを確認できます。
